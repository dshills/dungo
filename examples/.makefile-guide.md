### Building

```bash
make examples           # Build all examples to bin/ directory
```

### Running Individual Examples

```bash
make run-zelda         # Generate Zelda-style dungeon (20-25 rooms)
make run-metroidvania  # Generate Metroidvania maze (60-80 rooms)
make run-darksouls     # Generate Dark Souls challenge (30-40 rooms)
```

### Running Multiple Examples

```bash
make run-all-games     # Run all three game-style examples sequentially
```

### Technical Examples

```bash
make run-text-render   # Run text rendering example
make run-embedding     # Run embedding example
```

## ğŸ“ Output Locations

**Direct runs** (via `make run-*`):
- Output files created in each example's directory
- Example: `examples/01-zelda-dungeon/zelda_dungeon_*.{json,tmj,svg}`

**Binary runs** (via `bin/*`):
- Output files created in current working directory

## ğŸ’¡ Tips

- Use `make run-*` targets when developing/testing (no compilation needed)
- Use `make examples` to create distributable binaries
- Each example generates 3 files: `.json`, `.tmj`, and `.svg`
- Open `.svg` files in a browser to visualize dungeons
- Import `.tmj` files into Tiled Map Editor

## ğŸ”§ Customization

To customize dungeons, edit the `config.yaml` in each example directory, then run:

```bash
cd examples/01-zelda-dungeon
# Edit config.yaml
make run-zelda
```

Or use the binary with different configs:

```bash
make examples
cd examples/01-zelda-dungeon
../../bin/zelda-dungeon
```
